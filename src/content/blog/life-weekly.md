这一周的核心任务很明确：把个人博客从想法变成一个能打开的网站。

## 本周完成

- 安装并使用 GitHub CLI
- 创建 `hilavera-blog` 仓库
- 用 Astro 搭建博客项目
- 配置 GitHub Pages 自动部署
- 把首页改成个人空间布局

这些任务单独看都不复杂，但串起来之后就是一次完整的项目流程。

## 遇到的问题

### 1. 本地能打开，线上路径不对

部署到 GitHub Pages 时，仓库名会变成路径的一部分。页面里如果直接写 `/blog/`，线上访问就可能跳错地方。

解决办法是统一使用 `withBase()` 生成链接。

### 2. Chrome 插件安装不顺利

为了让开发助手能操作 Chrome，我需要安装 Codex Chrome 扩展。商店里一开始显示找不到商品，所以最后走了本地安装方式。

这件事让我意识到：开发环境本身也需要记录，不然后面换电脑会重新踩坑。

### 3. 首页内容太像模板

一开始的首页只是普通博客首页，不像个人空间，也不够像开发者主页。后来改成顶部个人名、左侧菜单、右侧内容面板，结构更清楚。

## 本周学到

这周最大的收获不是某个语法，而是项目闭环：

```text
想法 -> 本地实现 -> 构建检查 -> 推到 GitHub -> 线上验证
```

只有走完这个闭环，才算真正完成一次开发任务。
