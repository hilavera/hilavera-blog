这是这个博客的第一篇开发日志。

我现在还在入门阶段，很多东西不是“懂了”，只是“照着做能跑起来”。所以我想把过程写下来：哪里卡住了、怎么查资料、最后怎么解决。

## 为什么要公开记录

学习开发最容易遇到的问题是：当天感觉懂了，过几天又忘了。尤其是环境配置、部署路径、命令行报错这些内容，不写下来就很难复盘。

这个博客会记录三类内容：

- **学习路线**：当前在学什么，为什么先学这个
- **项目复盘**：做了什么功能，解决了什么问题
- **踩坑记录**：报错怎么出现，最后怎么定位

## 当前阶段

我的当前目标不是做复杂系统，而是先把基础打牢：

1. 能用 HTML 和 CSS 做出稳定页面
2. 能用 JavaScript 写简单交互
3. 能用 Git 和 GitHub 管理代码
4. 能把项目发布到线上

这四件事看起来基础，但真正做一遍会遇到很多细节。

## 博客会怎么更新

我会把每次改动写成一篇小文章。文章不一定长，但尽量完整：

- 这次要解决什么
- 改了哪些文件
- 遇到了什么问题
- 最后怎么验证

## 写给未来的自己

以后回头看时，我希望能看到自己是怎么一步步从“照着教程做”走到“能独立判断和实现”的。
