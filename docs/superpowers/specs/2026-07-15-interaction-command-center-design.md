# 博客交互中枢设计

## 目标

在保留当前顶部个人信息和左右两栏布局的前提下，让首页更有互动感和记忆点，同时不重新引入高成本视觉效果。

## 功能范围

### 1. 快速探索中枢

- 在个人信息区域加入明显的“快速探索”入口。
- 支持点击入口或按 `Command/Ctrl + K` 打开。
- 在一个输入框内搜索首页面板、文章分类和已发布文章。
- 支持方向键上下选择、回车确认和 `Escape` 关闭。
- 选择首页面板时原地切换，选择完整页面或文章时正常跳转。

### 2. 可操作的状态卡

- 将现有 NOW、LEARNING、NEXT 状态卡改为按钮。
- 每张卡片跳到最相关的首页面板或文章筛选结果。
- 保持当前卡片大小和状态栏布局不变。

### 3. 即时操作反馈

- 在访客互动区域增加分享和复制链接操作。
- 分享、复制、点赞、收藏和保存草稿后统一显示简短提示。
- 访客状态继续只保存在当前浏览器，并明确说明这不是公开评论系统。

### 4. 更完整的导航行为

- 在左侧标签栏内支持方向键左右切换。
- 当前面板继续同步到 URL 哈希。
- 浏览器前进和后退时无需刷新即可恢复对应面板。
- 从命令中枢或分类快捷入口切换面板后，将焦点移动到合适的位置。

## 实现结构

功能继续放在 `src/pages/index.astro`，因为它只属于首页且范围较小。Astro 根据现有导航、分类和文章数据生成精简的命令列表；一个内联脚本负责弹窗状态、结果过滤、键盘操作、面板切换、分享和提示反馈。

不增加第三方依赖。原生对话框、DOM、Web Share、Clipboard、History 和 `localStorage` API 足以完成需求。

## 数据流

1. Astro 从当前内容集合生成静态命令项。
2. 打开探索中枢后自动聚焦搜索输入框。
3. 输入内容按标题、描述、分类和标签过滤命令项。
4. 选择结果后调用现有面板/筛选函数，或者打开对应链接。
5. 本地互动先更新 `localStorage`，再通过提示区域反馈操作结果。

## 异常处理

- 浏览器不支持 Web Share 时，自动改为复制当前页面链接。
- Clipboard 不可用时，使用临时文本框完成复制降级。
- `localStorage` 不可用或数据损坏时，回退到新的内存状态。
- 没有搜索结果时显示空状态，不自动关闭探索中枢。

## 无障碍与性能

- 使用可访问的模态对话框、带标签的搜索框、当前结果状态和实时提示区域。
- 关闭后将焦点还给打开按钮。
- 遵守 `prefers-reduced-motion`。
- 只使用短暂、由操作触发的透明度和位移动效。
- 不增加背景模糊、滤镜、鼠标追踪、粒子或无限动画。

## 测试

- 实现前先增加会失败的测试，覆盖探索中枢结构、命令数据、状态快捷入口、分享/提示钩子以及键盘和哈希行为。
- 保留现有性能预算测试；如果出现新的高成本效果风险，再补对应限制。
- 运行完整构建和测试，最后在 Chrome 中检查桌面与移动端页面。

## 验收标准

- 首页原有布局和视觉系统保持可识别。
- 访客可以通过一个探索中枢找到面板、分类和文章。
- 鼠标和键盘流程都可以使用。
- 每个本地操作都有明确的视觉反馈。
- 现有性能预算测试继续通过。
- GitHub Pages 能在 `/hilavera-blog/` 基础路径下成功构建。

## 不包含

- 公开评论、用户账号、服务端点赞数量、统计分析和数据库。
- 全站重新设计或修改文章内容。
